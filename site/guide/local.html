local 模式

让本机 mixed 代理自动接入局域网里的上游网关。

local 模式不需要远端 tcptun server。它在本机开一个 mixed 代理端口，然后把需要走上游的流量转发到局域网网关代理。

适合场景

  • 路由器或旁路网关已经开了代理端口。
  • 同一局域网另一台机器运行 SOCKS5/mixed。
  • 你希望本机应用统一连 127.0.0.1:1080。

最小启动

tcptun local

默认行为：

  • 本机监听 127.0.0.1:1080。
  • 上游网关端口默认是 1080。
  • 上游协议默认是 socks5。
  • 会尝试自动发现默认网关 IP；如果不可连，会扫描本机所在内网 IPv4 网段。
  • 会读取 config.json 和 route.json，除非你显式传入空路径禁用。

固定网关 IP

如果你知道网关 IP，建议显式指定。这样不会扫描网段，启动更直接。

tcptun local \
  --listen 127.0.0.1:1080 \
  --gateway-ip 192.168.1.1 \
  --gateway-port 7890

参数含义：

  • --listen：本机代理监听地址。只给本机用时保持 127.0.0.1；要给局域网其他设备用才改成 0.0.0.0。
  • --gateway-ip：上游网关 IP，例如路由器、旁路由或代理机器。
  • --gateway-port：网关代理端口。常见是 1080、7890 或 10808。

上游协议：socks5 还是 mixed

值适合上游行为
socks5上游只支持 SOCKS5tcptun 会解析本地 HTTP/SOCKS5 请求，再转换成 SOCKS5 发给上游。未知 mixed 流量会被拒绝。
mixed上游同时支持 HTTP proxy 和 SOCKS5HTTP 代理和未知 mixed 流量会原样发给网关；SOCKS5 TCP/UDP 仍走 SOCKS5 协商。
 
tcptun local \
  --gateway-ip 192.168.1.1 \
  --gateway-port 7890 \
  --upstream-protocol mixed

本地 SOCKS5 入口加用户名密码

如果你把监听地址暴露给局域网，例如 0.0.0.0:1080，建议加认证。用户名或密码任一非空时，SOCKS5 会启用 username/password 认证。

tcptun local \
  --listen 0.0.0.0:1080 \
  --gateway-ip 192.168.1.1 \
  --gateway-port 7890 \
  --socks5-username alice \
  --socks5-password 'use-a-long-password'

HTTP proxy 入口不使用这组 SOCKS5 凭据；这组参数只控制本地 SOCKS5 客户端认证。

连接需要认证的上游 SOCKS5

如果局域网网关的 SOCKS5 端口本身要求用户名密码，用上游凭据参数。

tcptun local \
  --gateway-ip 192.168.1.1 \
  --gateway-port 1080 \
  --upstream-socks5-username gateway-user \
  --upstream-socks5-password 'gateway-password'

用 config.json 固化 local 配置

{
  "mode": "local",
  "listen_addrs": ["127.0.0.1:1080"],
  "upstream_protocol": "mixed",
  "socks5_username": "alice",
  "socks5_password": "use-a-long-password",
  "upstream_socks5_username": "gateway-user",
  "upstream_socks5_password": "gateway-password",
  "direct_probe_timeout": "500ms",
  "scan_retry_interval": "5s"
}

注意：gateway_ip、gateway_port、scan_workers、scan_timeout、refresh_interval 这些是 CLI/API 参数，不会从运行配置 JSON 加载，需要通过命令行传入。

网关发现和扫描怎么工作

  1. 检查本机是否有内网 IPv4 地址。
  2. 发现系统默认网关 IP。
  3. 尝试连接 <网关IP>:<gateway-port>。
  4. 如果失败，扫描本机内网 IPv4 网段，寻找打开了目标端口的主机。
  5. 如果找到多个候选，会按连接延迟排序，并让同一个源 IP 尽量保持使用同一个上游。
  6. 如果没有找到，会等待 --scan-retry-interval 后重试。
tcptun local \
  --gateway-port 7890 \
  --scan-timeout 200ms \
  --scan-workers 256 \
  --refresh-interval 5s \
  --scan-retry-interval 10s

性能建议

局域网大、机器多时可以提高 --scan-workers，但不要无限加大；扫描超时 --scan-timeout 过短会漏掉慢设备，过长会拖慢启动。

应用怎么使用

curl -x socks5h://127.0.0.1:1080 https://ifconfig.me
curl -x http://127.0.0.1:1080 https://example.com

浏览器里把 SOCKS5 或 HTTP 代理都设置到 127.0.0.1:1080。如果使用 SOCKS5，建议开启“通过代理解析 DNS”或使用支持远端解析的客户端。
