先选你的使用方式

local 模式：把局域网网关当上游

适合你的电脑和已有代理网关在同一个局域网里，例如路由器、旁路网关或另一台机器已经开了 SOCKS5/mixed 端口。tcptun 在本机开 mixed 代理，然后自动发现或扫描网关。

tcptun local \
  --listen 127.0.0.1:1080 \
  --gateway-port 7890

关键参数是 --gateway-ip、--gateway-port、--upstream-protocol、--route-config。

client/server 模式：自己搭一条隧道

适合你有一台 VPS 或远端服务器。server 在远端监听，client 在本机开 mixed 代理，浏览器或 CLI 请求进入本地端口后通过隧道到远端出站。

tcptun server --config server.json
tcptun client --config client.json

关键参数是 server_addr、token、tunnel_protocol、tunnel_transport、TLS 或 REALITY 字段。

从零开始的最短路径

1. 下载或构建

可以从 GitHub Releases 下载预编译包，也可以在源码目录构建。构建需要 Go 1.25 或更新版本。

make build
bin/tcptun version

2. 生成配置

第一次部署建议使用 tcptun config，它会写出 server.json、client.json 和 route.json。如果不带参数，会进入交互向导；带参数时直接生成，适合脚本化。

tcptun config \
  --protocol native \
  --transport raw \
  --server-addr proxy.example.com:9443 \
  --server-listen 0.0.0.0:9443 \
  --client-listen 127.0.0.1:1080

3. 把 server.json 放到服务器

远端服务器只需要 tcptun 二进制和 server.json。服务端默认会读取当前模式对应的 server.json，也可以显式指定路径。

tcptun server --config /etc/tcptun/server.json

4. 本机启动 client

本机保留 client.json 和 route.json，然后启动本地 mixed 代理端口。

tcptun client \
  --config ./client.json \
  --route-config ./route.json

5. 测试代理

SOCKS5 建议使用 socks5h，让域名解析也经过代理路径。

curl -x socks5h://127.0.0.1:1080 https://ifconfig.me
curl -x http://127.0.0.1:1080 https://example.com
