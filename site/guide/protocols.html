协议配置

native、VLESS、VMess、Trojan 每一种协议怎么配。

协议决定认证材料格式和兼容目标；transport 决定外层承载方式。自用优先 native，需要兼容 Xray/V2Ray 时选择 VLESS、VMess 或 Trojan。

一句话选择

自用性能和功能优先：native。REALITY/Vision：vless。VMess AEAD：vmess。Trojan 兼容：trojan。

native：自用推荐

native 是项目自己的隧道协议，适合同时控制 client 和 server 的场景。它支持 TCP、SOCKS5 UDP relay 和 tunnel mux。

tcptun config \
  --protocol native \
  --transport raw \
  --server-addr proxy.example.com:9443 \
  --server-listen 0.0.0.0:9443 \
  --client-listen 127.0.0.1:1080

命令行手动运行：

tcptun server \
  --listen 0.0.0.0:9443 \
  --tunnel-protocol native \
  --transport raw \
  --token change-me-to-a-long-random-token \
  --mux true

tcptun client \
  --listen 127.0.0.1:1080 \
  --server-addr proxy.example.com:9443 \
  --tunnel-protocol native \
  --transport raw \
  --token change-me-to-a-long-random-token \
  --mux true

VLESS：UUID 认证，可配 REALITY

VLESS 的 token 必须是 UUID。普通 VLESS 可以配 raw/ws/h2/h3；REALITY/Vision 只能用 VLESS + raw + reality。

tcptun config \
  --protocol vless \
  --transport raw \
  --server-addr proxy.example.com:9443 \
  --token 00000000-0000-4000-8000-000000000000
{
  "mode": "client",
  "listen_addrs": ["127.0.0.1:1080"],
  "server_addr": "proxy.example.com:9443",
  "token": "00000000-0000-4000-8000-000000000000",
  "tunnel_protocol": "vless",
  "tunnel_transport": "raw"
}

VMess：对接 VMess AEAD

VMess 的 token 也必须是 UUID。兼容目标是 VMess AEAD，security none。

tcptun server \
  --listen 0.0.0.0:9443 \
  --tunnel-protocol vmess \
  --transport raw \
  --token 00000000-0000-4000-8000-000000000000

tcptun client \
  --listen 127.0.0.1:1080 \
  --server-addr proxy.example.com:9443 \
  --tunnel-protocol vmess \
  --transport raw \
  --token 00000000-0000-4000-8000-000000000000

如果用配置生成器，留空 --token 会自动生成 UUID。

Trojan：password 认证，通常搭配 TLS

Trojan 的 token 是 Trojan password。生产环境建议使用 raw + TLS，并使用可信证书。

tcptun config \
  --protocol trojan \
  --transport raw \
  --tls \
  --tls-cert /etc/letsencrypt/live/proxy.example.com/fullchain.pem \
  --tls-key /etc/letsencrypt/live/proxy.example.com/privkey.pem \
  --tls-server-name proxy.example.com \
  --server-addr proxy.example.com:443 \
  --server-listen 0.0.0.0:443

生成后的服务端会写 tunnel_tls_cert 和 tunnel_tls_key；客户端会写 tunnel_tls: true 和 tunnel_tls_server_name。

常见错误

  • VLESS/VMess 的 token 不是 UUID：生成器会报 --token must be a UUID。
  • client 和 server 协议不一致：握手失败，连接无法建立。
  • client 和 server transport/path/TLS 不一致：看起来端口通，但应用请求失败。
  • 在非 native 协议上期待 tunnel mux：项目内 mux 只适合 native。
