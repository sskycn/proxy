tcptun
教程首页配置生成协议transport路由

client/server 模式

远端跑 server，本机跑 client，应用只连接本地代理端口。

这是自建隧道的标准部署方式。server 负责接收隧道并在远端连接真实目标；client 负责把本机 mixed 代理流量封装进隧道。

数据流

浏览器或 CLI → 127.0.0.1:1080 → tcptun client → 隧道 → tcptun server → 目标网站。

教程首页配置生成与文件local 网关模式client/server 模式协议配置transport、TLS、REALITY路由规则参数参考

1. 生成 native/raw 配置

把域名和端口换成你的服务器。没有域名时可以先写服务器 IP，例如 203.0.113.10:9443。

tcptun config \
  --protocol native \
  --transport raw \
  --server-addr proxy.example.com:9443 \
  --server-listen 0.0.0.0:9443 \
  --client-listen 127.0.0.1:1080 \
  --overwrite

2. 部署服务端

把二进制和 server.json 放到服务器，例如 /opt/tcptun/。

sudo mkdir -p /opt/tcptun
sudo cp tcptun /opt/tcptun/tcptun
sudo cp server.json /opt/tcptun/server.json
sudo chmod 755 /opt/tcptun/tcptun
sudo chmod 600 /opt/tcptun/server.json

前台测试运行：

cd /opt/tcptun
sudo ./tcptun server --config ./server.json --verbose

确认云防火墙和系统防火墙放行监听端口，例如 TCP 9443。

3. 用 systemd 常驻运行

[Unit]
Description=tcptun server
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
WorkingDirectory=/opt/tcptun
ExecStart=/opt/tcptun/tcptun server --config /opt/tcptun/server.json
Restart=on-failure
RestartSec=3
User=nobody
NoNewPrivileges=true

[Install]
WantedBy=multi-user.target
sudo install -m 0644 tcptun-server.service /etc/systemd/system/tcptun-server.service
sudo systemctl daemon-reload
sudo systemctl enable --now tcptun-server
sudo systemctl status tcptun-server

4. 本机启动 client

tcptun client \
  --config ./client.json \
  --route-config ./route.json \
  --verbose

也可以不用 JSON，全部从命令行传：

tcptun client \
  --listen 127.0.0.1:1080 \
  --server-addr proxy.example.com:9443 \
  --token change-me-to-the-same-token \
  --tunnel-protocol native \
  --transport raw

--token 必须和服务端一致；--tunnel-protocol、--transport、--tunnel-path、TLS/REALITY 参数也必须匹配。

5. 测试 TCP 和 HTTP 代理

curl -x socks5h://127.0.0.1:1080 https://ifconfig.me
curl -x http://127.0.0.1:1080 https://example.com
curl -v -x http://127.0.0.1:1080 https://www.google.com

如果使用浏览器，HTTP proxy 和 SOCKS5 proxy 都可以填 127.0.0.1、端口 1080。SOCKS5 更适合让域名也走代理解析。

服务端出站限制

tcptun server 只允许连接公网目标 IP。私有地址、回环地址、链路本地、组播、CGNAT 和保留网段会在拨号前被拒绝。这避免把公网服务端变成内网探测跳板。

如果你访问的是内网地址，应使用 local 模式或把服务端部署在那个内网里。

多监听地址

服务端和客户端都支持多个监听地址。命令行使用英文逗号分隔，JSON 使用数组。

tcptun server --listen 0.0.0.0:9443,[::]:9443 --config server.json
{
  "listen_addrs": ["0.0.0.0:9443", "[::]:9443"]
}

常见部署组合

性能优先

--protocol native
--transport raw
--mux true

自己控制两端时优先用这个。native 支持项目内 mux 和完整 UDP relay。

域名和 TLS

--transport raw --tls
--tls-cert fullchain.pem
--tls-key privkey.pem
--tls-server-name proxy.example.com

适合普通 HTTPS 端口部署，客户端会做证书校验。

nginx 后面

--transport ws
--tunnel-path /tcptun

server 监听 127.0.0.1，nginx 把 WebSocket 路径反代过去。

REALITY/Vision

--protocol vless
--transport raw
--tunnel-security reality
--flow xtls-rprx-vision

对接 VLESS REALITY/Vision 场景，参数见 transport 页面。
