transport 与安全层

raw、ws、h2、h3、TLS、REALITY 每种外层配置怎么填。

协议负责隧道内容，transport 负责把隧道放到 TCP、WebSocket、HTTP/2 或 HTTP/3 上。TLS 和 REALITY 是安全层，不要混淆。

选择建议

直连端口用 raw；nginx/CDN 后面用 ws；HTTP/2 基础设施用 h2；需要 QUIC 时用 h3；VLESS REALITY/Vision 用 raw + reality。

transport 对比

transport参数值适合场景必须注意
raw TCPraw最低开销、直连端口、TLS、Trojan、REALITY没有 HTTP 路径语义；防火墙需放行 TCP 端口。
WebSocketwsnginx、HTTP/1.1 反代、常见 CDNclient/server 的 tunnel_path 必须一致。
HTTP/2h2HTTP/2 upstream无证书时 server 可用 h2c；有证书时使用 TLS HTTP/2。
HTTP/3h3QUIC/HTTP/3server 必须配置证书和私钥；需要 UDP 端口可达。

raw：最低开销

tcptun config \
  --protocol native \
  --transport raw \
  --server-addr proxy.example.com:9443 \
  --server-listen 0.0.0.0:9443

raw 适合服务器直开端口。没有 TLS 时，外层就是普通 TCP；需要 TLS 时加 --tls、证书和 server name。

raw + TLS

服务端需要证书和私钥，客户端需要启用 TLS 并设置校验域名。

tcptun config \
  --protocol native \
  --transport raw \
  --tls \
  --tls-cert /etc/letsencrypt/live/proxy.example.com/fullchain.pem \
  --tls-key /etc/letsencrypt/live/proxy.example.com/privkey.pem \
  --tls-server-name proxy.example.com \
  --server-addr proxy.example.com:443 \
  --server-listen 0.0.0.0:443

只在测试时使用 --tls-insecure，它会让客户端跳过证书校验。

WebSocket：放在 nginx 后面

tcptun server 监听本机回环地址，nginx 对外监听 443 并把指定路径反代给 tcptun。

tcptun server \
  --listen 127.0.0.1:9443 \
  --tunnel-protocol native \
  --transport ws \
  --tunnel-path /tcptun \
  --token change-me

tcptun client \
  --listen 127.0.0.1:1080 \
  --server-addr proxy.example.com:443 \
  --tunnel-protocol native \
  --transport ws \
  --tunnel-path /tcptun \
  --tls \
  --tls-server-name proxy.example.com \
  --token change-me
location /tcptun {
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header Host $host;
    proxy_pass http://127.0.0.1:9443;
}

--tunnel-path 必须以 / 开头。生成器会自动把 tcptun 规范成 /tcptun。

HTTP/2

tcptun server \
  --listen 127.0.0.1:9443 \
  --transport h2 \
  --tunnel-path /tcptun \
  --token change-me

tcptun client \
  --server-addr 127.0.0.1:9443 \
  --transport h2 \
  --tunnel-path /tcptun \
  --token change-me

如果配置证书，server 使用 TLS HTTP/2；没有证书时可用于 h2c 场景。

HTTP/3

HTTP/3 基于 QUIC，服务端需要证书和私钥，防火墙要放行对应 UDP 端口。

tcptun server \
  --listen 0.0.0.0:9443 \
  --transport h3 \
  --tunnel-path /tcptun \
  --tls-cert /etc/letsencrypt/live/proxy.example.com/fullchain.pem \
  --tls-key /etc/letsencrypt/live/proxy.example.com/privkey.pem \
  --token change-me

tcptun client \
  --server-addr proxy.example.com:9443 \
  --transport h3 \
  --tunnel-path /tcptun \
  --token change-me

VLESS REALITY/Vision

REALITY 是 VLESS 的安全层配置，不是普通 TLS。它需要 --protocol vless、--transport raw、--tunnel-security reality。不能和 --tls 同时使用。

tcptun config \
  --protocol vless \
  --transport raw \
  --tunnel-security reality \
  --flow xtls-rprx-vision \
  --server-addr proxy.example.com:443 \
  --server-listen 0.0.0.0:443 \
  --reality-server-name www.example.com \
  --reality-server-names www.example.com \
  --reality-fingerprint chrome \
  --reality-short-id '' \
  --reality-short-ids '' \
  --reality-dest www.example.com:443 \
  --reality-spider-x /

字段解释：

  • reality_private_key：服务端私钥。生成 both/server 且留空时会自动生成。
  • reality_public_key：客户端公钥。生成器可由 private key 推导；只生成 client 时必须提供。
  • reality_server_name：客户端发出的 serverName。
  • reality_server_names：服务端允许的 serverName 列表。
  • reality_short_id 与 reality_short_ids：客户端单个 shortId 与服务端允许列表，十六进制字符串，允许为空。
  • reality_dest：REALITY fallback 目标，格式 host:port。
  • reality_fingerprint：客户端 uTLS 指纹，例如 chrome。

密钥安全

不要把真实 UUID、REALITY private key、public key 或服务器地址提交进公开仓库。文档里只使用占位值。
