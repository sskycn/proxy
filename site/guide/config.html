配置生成

用 tcptun config 生成可直接运行的三份配置。

生成器会根据你选择的协议、transport、安全层和监听地址写出服务端、客户端和路由文件。它也会自动生成 token、UUID 或 REALITY 密钥。

本页会生成

  • server.json：给远端 tcptun server。
  • client.json：给本机 tcptun client。
  • route.json：给 client/local 做强制上游路由。

最推荐的生成方式

如果你第一次使用，先用 native + raw 得到一套最小配置。把 proxy.example.com 改成你的域名或服务器 IP。

tcptun config \
  --protocol native \
  --transport raw \
  --server-addr proxy.example.com:9443 \
  --server-listen 0.0.0.0:9443 \
  --client-listen 127.0.0.1:1080 \
  --overwrite

参数解释：

  • --protocol native：使用 tcptun 原生协议，功能最完整，支持 TCP、SOCKS5 UDP relay 和 tunnel mux。
  • --transport raw：隧道直接跑在 TCP 上，额外开销最低。
  • --server-addr proxy.example.com:9443：写入 client.json，客户端会连接这个地址。
  • --server-listen 0.0.0.0:9443：写入 server.json，服务端监听所有 IPv4 地址的 9443 端口。
  • --client-listen 127.0.0.1:1080：写入 client.json，本机应用连接这个本地 mixed 代理。
  • --overwrite：如果目标文件已经存在，允许覆盖。生产环境覆盖前请备份旧配置。

交互模式

不带任何生成参数时，命令会进入交互向导。每个问题方括号内是默认值，直接回车会使用默认值。

tcptun config

典型选择流程：

  1. Protocol 选 native、vless、vmess 或 trojan。
  2. Config target 选 both 会同时生成 server、client、route。
  3. Tunnel transport 选 raw、ws、h2 或 h3。
  4. Server address for client config 填公网域名或 IP，例如 proxy.example.com:443。
  5. Shared token/password/UUID 留空时自动生成。VLESS/VMess 会生成 UUID，native/Trojan 会生成随机十六进制 token。
  6. 如果是 VLESS 并选择 REALITY，按提示填 serverName、dest、shortId 等字段。

生成 target 怎么选

参数输出什么时候用
--target bothserver.json、client.json、route.json默认值。自己同时控制服务端和客户端时使用。
--target server只写服务端配置只需要部署远端服务端，或客户端配置由别处提供。
--target client写客户端配置和路由配置连接已有服务端，例如已有 Xray VLESS REALITY 参数。
 
tcptun config \
  --target client \
  --output client.json \
  --route-output route.json \
  --protocol vless \
  --server-addr proxy.example.com:443 \
  --token 00000000-0000-4000-8000-000000000000

输出路径和覆盖规则

默认输出到当前目录。你可以用 --out-dir 指定目录，也可以单独设置每个文件名。

tcptun config \
  --out-dir ./deploy \
  --server-output server.json \
  --client-output laptop-client.json \
  --route-output laptop-route.json \
  --server-addr proxy.example.com:9443

如果文件已存在且没有 --overwrite，命令会报错并停止。这是为了避免误覆盖密钥、token 和路由学习结果。

配置文件字段示例

server.json

{
  "mode": "server",
  "listen_addrs": ["0.0.0.0:9443"],
  "token": "change-me-to-a-long-random-token",
  "tunnel_protocol": "native",
  "tunnel_transport": "raw",
  "tunnel_path": "/proxy"
}

client.json

{
  "mode": "client",
  "listen_addrs": ["127.0.0.1:1080"],
  "server_addr": "proxy.example.com:9443",
  "token": "change-me-to-a-long-random-token",
  "tunnel_protocol": "native",
  "tunnel_transport": "raw",
  "tunnel_path": "/proxy",
  "upstream_protocol": "socks5"
}

route.json

{
  "force_upstream": {
    "domains": [],
    "domain_regexes": [],
    "domain_suffixes": [],
    "ip_cidrs": [],
    "ip_ranges": [],
    "ips": []
  }
}

每类生成参数怎么填

类别参数具体填法
协议--protocolnative 自用优先；vless 对接 VLESS/REALITY；vmess 对接 VMess AEAD；trojan 对接 Trojan。
transport--transportraw 最低开销；ws 适合 nginx/CDN；h2 适合 HTTP/2；h3 适合 QUIC/HTTP/3。
监听--server-listen、--client-listen多个地址用英文逗号分隔，例如 0.0.0.0:443,[::]:443。
认证--tokenVLESS/VMess 必须是 UUID；native/Trojan 可以是长随机字符串。留空自动生成。
TLS--tls、--tls-cert、--tls-key、--tls-server-name服务端写证书和私钥路径，客户端写启用 TLS 和校验名称。
REALITY--tunnel-security reality 及 --reality-*只能配合 --protocol vless --transport raw。生成 both/server 时 private key 可留空自动生成。
mux--mux true|falsetcptun 两端部署时建议开启；VLESS/VMess 使用 Xray 兼容 mux.cool，Trojan 使用 tcptun 私有 mux。
本地认证--client-socks5-username、--client-socks5-password写入客户端本地 SOCKS5 入口认证；用户名或密码任一非空都会启用认证。
路由--force-ip-cidrs生成 route.json 时预置强制上游 CIDR，例如 1.1.1.0/24,8.8.8.8/32。
